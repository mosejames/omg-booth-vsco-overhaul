Phase 1: Backup & Prep

4 steps
~5 hrs
Step 1
Screenshot every existing questionnaire
Open each of the 28 questionnaires. Screenshot the full field list for each one. Save to a folder called "VSCO Backup." This is your safety net — if anything goes wrong, you can rebuild from these.
Settings › Questionnaires › [each form]
2-3 hrs
Step 2
Create the "Qualified" lead status
Add a new lead status called "Qualified". This is the trigger for the entire automation chain — when you mark a lead as Qualified, it kicks off the Event Details form, SMS, and (later) AI emails.
Settings › Lead Sources › Lead Statuses
15 min
Step 3
Create the "contact-preference" custom field
Create a new custom field. Name: Contact Preference. Type: Dropdown. Values: Email, Text, Both. Token will be {{job.custom.contact-preference}}. This controls whether a client gets email, SMS, or both.
Settings › Custom Fields › New Custom Field
15 min
Step 4
Audit and create other custom fields
Review the 99 existing custom fields. Delete any that are clearly unused. Create new ones you'll need for the forms: venue-name, venue-address, guest-count, event-vibe, guest-type, priorities. Each will have a token like {{job.custom.venue-name}}.
Settings › Custom Fields
Don't delete a custom field if you're not 100% sure it's unused — deleting removes it from all historical jobs too.
2-3 hrs

5 steps

Create three new products. Each one needs a name, price, description, and product image (required for Upgradable quote cards).

Phase 3: Build the Forms

6 steps
~45 hrs
Step 10
Build Form 1 — New Lead Intake
Create a new questionnaire. Set visibility to Public. Enable auto-create lead. Set a redirect URL to your thank-you page.
  • First Name, Last Name, Email, Phone (required)
  • Event Type — dropdown linked to {{job.type}} (Corporate, Wedding, Birthday, Expo / Trade Show, Multi-City Tour, School / Education, Virtual Experience, etc.)
  • Event Date — date picker
  • Estimated Guest Count — dropdown ranges (Under 50, 50-100, 100-200, 200-500, 500+)
  • Budget Range — dropdown ($1,200-$2,500, $2,500-$4,000, $4,000-$7,000, $7,000-$15,000, $15,000+, Not sure yet)
  • How did you hear about us? — dropdown (Referral, Saw us at an event, Instagram, Google, Event planner, Other)
  • What should guests walk away remembering? — open text
  • Preferred Contact Method — dropdown linked to {{job.custom.contact-preference}} (Email, Text, Both)
Settings › Questionnaires › New Questionnaire
This form will be embedded on your website. Keep it under 90 seconds to complete. See Client Journey Step 1 for the suggested copy and layout.
4-6 hrs
Step 11
Build Form 2 — Event Details
Consolidates 11 "Event Customization" variants and "Step 2 of 2" into one form. Set visibility to Private (sent via automation after qualifying).
  • Event Name, Venue Name, Venue Address, Start/End Time
  • Color palette, theme/style, dress code
  • Who are your guests? — dropdown (Employees, Clients/Customers, Friends & Family, VIPs, General Public, Mixed)
  • What matters most? (pick 2) — multi-select (Guest Experience, Social Media Content, Custom Branding, Print Keepsakes, Data & Analytics, Wow Factor)
  • What's the vibe? — dropdown + open text (Elegant/Formal, Fun/Playful, Modern/Minimalist, Bold/Energetic, Glamorous, Casual)
  • Any inspiration? — optional open text
  • Special requests or must-haves — open text
Settings › Questionnaires › New Questionnaire
12-18 hrs
Step 12
Build Form 3 — Event Logistics
Use the existing "Event Logistics" form (402 responses) as a base. Merge in the 4 type-specific variants and 2 virtual booth forms. Set visibility to Private. This form gets embedded in the Upgradable Quote, not sent separately.
  • Load-in time, setup location, power access, WiFi details
  • Venue contact name and number
  • Parking/loading dock instructions
  • Any venue restrictions (noise, flash, props)
Settings › Questionnaires › Edit existing "Event Logistics"
10-14 hrs
Step 13
Build Form 4 — Design & Branding Brief
New form. Absorbs "Design Assets Review" (1 response). Set to Private. Sent 21 days before event via automation.
  • Upload logo(s) — file upload field
  • Upload brand guidelines — file upload (optional)
  • Brand colors (hex codes or description)
  • Overlay/template preferences
  • Social handles to display
  • Any specific text for prints
Settings › Questionnaires › New Questionnaire
6-8 hrs
Step 14
Update Form 5 — Pre-Event Report
This form is battle-tested (101 responses). Clean up field names for consistency, set visibility to Private. This is an internal form — your team fills it out 2 days before the event.
Settings › Questionnaires › Edit existing form
2-3 hrs
Step 15
Build Form 6 — Post-Event Report
Merge the EM version (139 responses) and Smile Squad version (65 responses) into a single form with a role selector at the top (Event Manager / Smile Squad). Set to Private. Sent 1 day after event via automation.
  • Role selector (determines which fields show)
  • Event summary, guest engagement notes, equipment issues
  • Photo/video count, social media metrics
  • Client feedback received, suggestions for improvement
Settings › Questionnaires › New Questionnaire
6-10 hrs

Step 27
Build 5 core SMS automations
Create these SMS automations in VSCO. All conditioned on {{job.custom.contact-preference}} = "Text" or "Both":
  • SMS 2 — Qualified Nudge: Trigger = Lead status → Qualified. "Hey {{name}}, just sent over some info about your event. Check your inbox!"
  • SMS 4 — Proposal Ready: Trigger = Quote sent. "Your custom proposal is in your inbox. Take a look when you're ready!"
  • SMS 5 — Booking Confirmation: Trigger = Payment received. "You're officially booked! Check your email for all the details."
  • SMS 6 — Design Brief Reminder: Trigger = 14 days before event. "Hey {{name}}, don't forget to send your logos. Check your email for the design brief!"
  • SMS 8 — Day-Of Arrival: Trigger = Day of event, 8am. "Today's the day! Our team is heading your way. See you soon!"

Step 31
Test the full flow with a dummy lead
Create a test lead and run through every step:

  • Verify lead is created in VSCO with correct job type
  • Mark lead as "Qualified" — verify Form 2 email sends

Step 42
Build Make.com Scenario 1: Draft Generation
The AI pipeline. Create a new scenario in Make.com:
  • Module 1: Webhook (custom) — copy the URL for later
  • Module 2: HTTP Request → POST to https://api.anthropic.com/v1/messages with your system prompt + lead data. Model: claude-haiku-4-5-20251001
  • Module 3: Google Sheets → Add Row (log email draft, SMS draft, client info, status = "Pending")
  • Module 4: Gmail → Send email to yourself with both drafts + Approve/Edit buttons
make.com › Scenarios › New Scenario
Paste the system prompt from the AI Integration Plan Section 05.

Step 43

  • Module 1: Webhook (triggered by your Approve button click)
  • Module 2: Google Sheets → Read the draft row
  • Module 3: Gmail → Send the approved email to the client
  • Module 4: Router → If contact preference = "Text" or "Both"...
  • Module 4a: HTTP Request → POST to Quo API to send SMS
  • Module 5: Google Sheets → Update status to "Sent"
make.com › Scenarios › New Scenario

Step 44
Add Post Web Request to "Qualified" automation
Go to the automation you built in Step 21. Add a second action: Post Web Request. Paste your Make.com Scenario 1 webhook URL. Now when you qualify a lead, VSCO sends the template email (Form 2) AND triggers the AI pipeline simultaneously.
Settings › Automations › [Qualified automation]
15 min
Step 45
Test the full AI + SMS flow
  • Create a test lead with contact preference = "Both"
  • Fill out Forms 1 and 2 with realistic data
  • Mark lead as "Qualified"
  • Check: Did Make.com receive the webhook? (Execution history)
  • Check: Did Claude generate a good email draft AND SMS draft? (Google Sheet)
  • Check: Did you receive the notification email on your phone?
  • Tap "Approve" — did the client receive the email?
  • Did the SMS send via Quo?
  • Is the Google Sheet status updated to "Sent"?
Full AI pipeline test
If drafts don't sound right, adjust the system prompt in Make.com Scenario 1 Module 2. Most tuning happens here.
2-3 hrs
Step 46
Iterate on the system prompt
Run 5-10 real leads through the AI pipeline. Review every draft. Adjust the system prompt until you're approving 90%+ without edits. Common tweaks: tone too formal, too casual, too long, not referencing enough details, recommending wrong package tier.
make.com › Scenario 1 › Module 2 (system prompt field)
Keep a note of which drafts you edited and why. Patterns in your edits tell you what to fix in the prompt.
2-3 hrs
