Standard Operating Procedure

6-Week Build Schedule

The entire overhaul broken into 18 focused sessions across 6 weeks. Three sessions per week (Mon / Wed / Fri), ~1.5 hours each. Check off tasks as you go. Progress saves in your browser.

Week 1
0 / 18 sessions
Ground Rules
  • ~1.5 hours per session. No marathons. Consistency beats intensity. Close the laptop when the session is done.
  • Three sessions per week: Monday, Wednesday, Friday. 18 sessions total. If you miss a day, slide it. Don't double up.
  • Don't skip ahead. Each session builds on the one before it. The order matters.
  • Test as you go. After building anything (form, automation, product), test it immediately. Don't batch testing for the end.
  • Screenshot before you delete. Every time. No exceptions.
  • If something takes longer than expected, finish it next session. Don't rush. The timeline flexes. The quality doesn't.
CC

Claude Code Acceleration

Throughout this schedule, you'll see Claude Code Assist badges on tasks where Claude Code can do the heavy lifting. Instead of writing email templates, SMS copy, product descriptions, and CSS from scratch, you ask Claude to draft them in your brand voice. You review, tweak, paste into VSCO.

This turns 2-hour writing sessions into 20-minute review sessions. Eight of the 18 sessions have Claude Code acceleration built in.
