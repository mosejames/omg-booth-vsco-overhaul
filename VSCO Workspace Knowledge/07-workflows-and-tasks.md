# VSCO Workspace - Workflows and Tasks

## Workflows

**Location:** Settings > Workflow

A Workflow is the set of tasks you complete throughout the process of taking a client from new lead to completed job.

### Key Features:
- Create task lists with individual tasks for each job stage (Lead, Booked, Fulfillment, Completed)
- Workflows are attached to Job Types -- when a new lead/booking is created with that job type, workflow tasks are auto-assigned
- Tasks can be automatically assigned to a specific employee
- Task due dates can be set relative to an event on the job or a nearby task
- When jobs are rescheduled, task due dates based on session dates automatically recalculate
- Install pre-built workflows from industry professionals with one click, then customize

### Template Gallery:
- Browse industry-specific workflow templates
- Install full workflows designed by professionals
- Customize after installation to match your business

---

## Deliverable Workflows

A separate concept from main workflows:
- A deliverable workflow is a list of tasks tied to a specific product
- Tasks only appear on a job when that product is ordered
- Attached when creating products in Settings > Products & Services
- Example: Album design tasks only appear when an album product is ordered

---

## Task Lists vs. Tasks

### Task Lists
- Subcategories/groupings of tasks -- NOT the tasks themselves
- Each task list must have at least one task under it
- All tasks live under task lists
- Drag and drop to reorder task lists between sections

### Tasks
- Individual action items within a task list
- Can be automatically assigned to an employee
- Can have due dates set relative to events or nearby tasks
- Due dates recalculate automatically when jobs are rescheduled
- Drag and drop to reorder within or between task lists

---

## Task Completion Triggers

When you check off a task, it can trigger automations using the **Task > Completed** observable:
- Send an email to a client when a task is ticked off
- Create another follow-up task
- Update a field
- Chain tasks together for sequential workflows

---

## Add Task Action (Automation)

When adding a task as an automation action, you configure:
- **Task List name** -- which task list it belongs to
- **Task name** -- the name of the task
- **Due date** -- set relative to the original observable event
- **Assignee** -- assign to a specific VSCO Workspace user
